# 8 月案例训练手册

目标：把数据开发经验转换成系统架构设计师案例题答案。

## 案例题答题框架

遇到案例题先按这 5 步组织：

1. 识别系统目标：业务目标、用户规模、数据规模、时效要求。
2. 识别质量属性：性能、可用性、可靠性、安全性、可扩展性、可维护性。
3. 找瓶颈与风险：单点、数据倾斜、延迟、重复消费、权限缺失、容量不足。
4. 给架构方案：分层、异步、缓存、限流、分区、冗余、监控、容灾。
5. 写效果与权衡：提升了什么，引入了什么成本，如何验证。

## 数据开发专项素材

### 实时计算架构

常见问题：数据延迟高、Kafka 积压、Flink 反压、checkpoint 失败、下游写入慢、重复消费或数据丢失。

常用答案点：

- 提高并行度，按 key 合理分区，处理数据倾斜。
- 调整 checkpoint 间隔、超时、状态后端和 RocksDB 参数。
- 下游写入使用批量、异步、连接池、幂等键。
- 使用监控告警观察 lag、反压、失败率、端到端延迟。
- 对关键链路设计重放、补偿、死信队列和数据校验。

### 数据库与 SQL 优化

常见问题：查询慢、大表 join 成本高、索引失效、锁冲突、报表影响在线业务。

常用答案点：

- 查看执行计划，优化索引、连接顺序、过滤条件和字段选择。
- 对大表进行分区、分库分表、冷热分离。
- 建立汇总表、物化视图或离线预计算。
- 读写分离，报表库与在线库隔离。
- 对高频 SQL 做限流、缓存和慢 SQL 治理。

### 数据平台高可用

常见问题：调度系统单点故障、数据链路中断无法发现、上游接口波动导致全链路失败、数据质量问题晚发现。

常用答案点：

- 调度服务集群化，关键组件主备或多副本。
- 任务失败自动重试，超过阈值告警。
- 设计数据质量校验：完整性、唯一性、及时性、准确性。
- 保留中间层数据，支持重跑和补数。
- 监控任务成功率、延迟、吞吐、积压、资源使用率。

### 安全与治理

常见问题：敏感数据暴露、指标口径不一致、权限过粗、数据血缘不清。

常用答案点：

- 分级分类、脱敏、加密、最小权限、操作审计。
- 建立指标字典、元数据管理、血缘追踪。
- 对数据访问做审批、授权、日志留存。
- 建立数据质量规则和异常处理流程。

## 每周训练方式

- 周一至周五：每天拆 1 道案例小题，写 5-8 个要点。
- 周六：限时完成 1 道完整案例。
- 周日：复盘答案结构，把可复用句子加入素材库。

## 案例阶段验收

- 至少完成 8 道案例题训练。
- 能熟练写出实时计算、数据库优化、高可用、安全治理 4 类题目的答案框架。
- 每类题目都有 10 条以上可复用答案句。
- 案例小模拟能在规定时间内完成，不出现大段空白。
